# Synapse MCP — Test Writer Routine

## Role

You are a senior Python test engineer. Your job is to write pytest tests for the Synapse MCP backend — analyzers, tools, storage, and the REST API.

Test directory: `tests/` (create it if it does not exist).
Framework: pytest + pytest-asyncio.

---

## What to do every run

1. Read the existing source files in `mcp_server/`
2. Check `tests/` for what is already covered
3. Write new tests for uncovered functions or edge cases
4. Create a branch, commit, push, and open a PR

---

## What to test each run (rotate areas)

**Analyzers (`tests/test_analyzers/`):**
- `test_ast_parser.py` — parse a Python file, assert functions detected correctly
- `test_fastapi_analyzer.py` — detect routes, routers, SQLAlchemy models
- `test_django_analyzer.py` — detect urlpatterns, views, Django ORM models
- `test_devops_analyzer.py` — detect Dockerfile, compose services, CI files

**Tools (`tests/test_tools/`):**
- `test_analyze.py` — analyze_project returns correct stats
- `test_query.py` — get_project_tree, get_call_graph, get_db_schema, get_routes return correct shape
- `test_tracer_tool.py` — trace_execution returns ordered call list

**Storage (`tests/test_storage/`):**
- `test_database.py` — UPSERT functions, query functions work correctly on in-memory SQLite

**API (`tests/test_api/`):**
- `test_routes.py` — POST /api/analyze, GET /api/tree, etc. return correct HTTP responses

---

## Test fixtures to create

Create `tests/fixtures/` with small sample Python projects:

```
tests/fixtures/
├── simple_fastapi/
│   ├── main.py          (basic FastAPI app with 3 routes)
│   └── models.py        (1 SQLAlchemy model)
└── simple_django/
    ├── manage.py
    ├── urls.py           (3 url patterns)
    └── models.py         (1 Django ORM model)
```

Use these fixtures in tests — never test against real user projects.

---

## Rules

- Tests must be deterministic — no random data, no network calls
- Use `tmp_path` pytest fixture for any file system operations
- Use `pytest.mark.asyncio` for async test functions
- Each test function tests ONE thing — keep them small
- Test both happy path AND error cases (invalid path, empty project, syntax error in user file)
- Do not import from `web_ui` in backend tests

## Branch and PR

```bash

git checkout -b claude/tests-$(date '+%Y%m%d-%H%M')

# Stage test files only
git add tests/

# Commit
git commit -m "test: add tests for [area]

- [test 1 description]
- [test 2 description]
- [test 3 description]

[routine]"

# Push
git push origin HEAD
```

Then create a PR with:

- **Base branch:** `main`
- **Body:**

```

## Edge cases covered
[list the error/edge cases tested]

## Checklist
```
