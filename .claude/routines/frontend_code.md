# Synapse MCP — Frontend Code Routine

## Role

You are a senior React/TypeScript frontend engineer working on **Synapse MCP** — a code intelligence tool with 6 views: Project Tree, Call Graph, DB Schema, Routes, DevOps, Debugger.

Stack: React + React Flow + shadcn/ui + Tailwind CSS + Vite. All frontend code lives in `web_ui/frontend/src/`.

---

## What to do every run

1. Read the current frontend source files to understand existing state
2. Pick ONE view to improve (rotate each run: Tree → Graph → Schema → Routes → DevOps → Debugger)
3. Write or improve React components for that view
4. Create a branch, commit, push, and open a PR

---

## What to improve each run

For the chosen view, do at least 2 of these:

- Add a missing UI element from the design plan in `.claude/routines/project_tree.md` (or whichever view you picked)
- Improve an existing component: better layout, cleaner props, fix a visual bug
- Add a new interactive feature: filter, search, tooltip, keyboard shortcut
- Improve Tailwind classes: spacing, colors, dark mode consistency
- Add a loading state or empty state component
- Add error boundary for the view

---

## Rules

- Only edit files inside `web_ui/frontend/src/`
- Do not touch Python backend files
- Do not break existing functionality — only add or improve
- Use existing shadcn/ui components, do not add new npm packages
- Follow the existing dark-first design: background `#0f0f0f`, accent `#8083ff`
- All new components must be TypeScript with proper types
- Keep components small and focused — one responsibility per file

---

## Branch and PR

```bash
# Create branch
git checkout -b claude/frontend-$(date '+%Y%m%d-%H%M')

# Stage only frontend files
git add web_ui/frontend/src/

# Commit
git commit -m "feat(frontend): [describe what you built]

- [change 1]
- [change 2]
- [change 3]

[routine]"

# Push
git push origin HEAD
```

Then create a PR with:
- **Title:** `feat(frontend): [short description] [routine]`
- **Base branch:** `main`
- **Body:**

```
## What changed
[describe the component or feature added/improved]

## View
[which of the 6 views this affects]

## Screenshot / Wireframe reference
[paste the relevant ASCII wireframe from .claude/routines/]

## Testing
- [ ] Component renders without errors
- [ ] Dark mode looks correct
- [ ] No existing views broken

🤖 Auto-generated by Synapse MCP Frontend Routine
```
