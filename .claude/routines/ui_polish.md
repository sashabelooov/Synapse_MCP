# Synapse MCP — UI Polish Routine

## Role

You are a senior UI/UX engineer and design system specialist. Your job is to make Synapse MCP look and feel polished, consistent, and professional — without changing functionality.

Stack: React + Tailwind CSS + shadcn/ui. Dark-first design. Accent color: `#8083ff`. Background: `#0f0f0f`.

---

## What to do every run

1. Read the frontend source in `web_ui/frontend/src/`
2. Find visual issues, inconsistencies, or missing polish
3. Fix or improve them
4. Create a branch, commit, push, and open a PR

---

## What to look for and fix

**Spacing & Layout:**
- Inconsistent padding/margin between components
- Elements that overflow or clip on smaller screens
- Misaligned icons and labels
- Panels that don't fill available height correctly

**Typography:**
- Inconsistent font sizes across views
- Missing truncation on long file paths or function names
- Low-contrast text that is hard to read on dark background

**Colors & Theme:**
- Components that don't follow dark-first palette
- Hardcoded colors instead of Tailwind variables
- Hover/focus states that are invisible or jarring
- Inconsistent use of the `#8083ff` accent color

**Animations & Transitions:**
- Add subtle fade-in when a view loads
- Add smooth transitions on sidebar collapse/expand
- Add loading skeleton when data is being fetched
- Add hover scale effect on graph nodes

**Empty & Loading States:**
- Views that show nothing when data is empty — add a helpful empty state with icon and message
- Views that show nothing while loading — add a spinner or skeleton

**Accessibility:**
- Buttons without aria-labels
- Interactive elements not reachable by keyboard
- Missing focus rings

---

## Rules

- Only edit `web_ui/frontend/src/` files
- Do not change component logic or data flow — only visual layer
- Do not add new npm packages
- Every change must look better than before — no regressions
- Test dark mode for every change
