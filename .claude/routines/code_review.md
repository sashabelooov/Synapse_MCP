# Synapse MCP — Code Review Routine

## Role

You are a senior code reviewer. Your job is to review the most recent commits on `main`, find real bugs, security issues, and code quality problems, then fix them.

---

## What to do every run

1. Run `git log --oneline -10` to see recent commits
2. Run `git diff HEAD~5 HEAD` to see what changed in the last 5 commits
3. Read the changed files carefully
4. Find real problems — bugs, edge cases not handled, security issues, bad error handling, performance problems
5. Fix the problems directly in code
6. Create a branch, commit, push, and open a PR

---

## What to look for

**Bugs:**
- Function that can crash with a KeyError, AttributeError, or IndexError
- Async function called without await
- SQLite query that can return None but code doesn't handle it
- File path that assumes Linux but won't work on Windows

**Security:**
- Path traversal: user-supplied path not sanitized before `open()` or `os.walk()`
- Command injection: user input passed to `subprocess` without sanitization
- No file size check before reading — could read a giant file into memory

**Error handling:**
- Bare `except:` that swallows all errors silently
- Function that returns `None` when caller expects a dict
- Missing `{"ok": false, "error": "..."}` response on failure paths

**Code quality:**
- Dead code: imported but never used
- Duplicate logic that should be a shared function
- Magic numbers or hardcoded strings that should be constants
- Function longer than 60 lines that should be split

**Performance:**
- N+1 database queries in a loop — should be one query
- Reading an entire file when only the first 100 lines are needed
- Rebuilding a graph from scratch when only a delta changed

---

## Rules

- Only fix real problems — do not refactor for style
- Do not change function signatures that MCP tools depend on
- Do not change the REST API response shape
- Each fix must be a targeted, minimal change — not a rewrite
- If you find a security issue (path traversal, injection), fix it immediately and mark it `[SECURITY]` in the commit

---

## Branch and PR

```bash
# Create branch
git checkout -b claude/review-$(date '+%Y%m%d-%H%M')

# Stage fixed files
git add [files you fixed]

# Commit
git commit -m "fix: [describe the bugs fixed]

- [fix 1] in [file:line]
- [fix 2] in [file:line]
- [fix 3] in [file:line]

Reviewed commits: [list commit hashes reviewed]
[routine]"

# Push
git push origin HEAD
```

Then create a PR with:
- **Title:** `fix: code review findings [routine]`
- **Base branch:** `main`
- **Body:**

```
## Commits reviewed
[list the commit hashes and messages reviewed]

## Bugs found and fixed
| File | Line | Issue | Fix |
|------|------|-------|-----|
| [file] | [line] | [what was wrong] | [how it was fixed] |

## Security issues
[list any security fixes, or "None found"]

## Checklist
- [ ] All fixes are minimal and targeted
- [ ] No MCP tool signatures changed
- [ ] No API response shapes changed

🤖 Auto-generated by Synapse MCP Code Review Routine
```
