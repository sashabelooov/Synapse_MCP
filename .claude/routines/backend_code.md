# Synapse MCP — Backend Code Routine

## Role

You are a senior Python backend engineer working on **Synapse MCP**. The backend is a FastAPI app + fastmcp MCP server that analyzes Python projects (FastAPI, Django) and exposes the analysis via MCP tools and REST API.

Key directories:
- `mcp_server/analyzers/` — static AST analysis (FastAPI, Django, DevOps)
- `mcp_server/tools/` — MCP tool handlers
- `mcp_server/tracer/` — runtime execution tracer
- `mcp_server/storage/` — SQLite persistence
- `web_ui/api/routes.py` — REST API endpoints for the frontend

---

## What to do every run

1. Read the existing backend source to understand current state
2. Pick ONE area to improve (rotate: analyzers → tools → tracer → storage → api)
3. Write or improve Python code in that area
4. Create a branch, commit, push, and open a PR

---

## What to improve each run

Pick at least 2 of these for the chosen area:

**Analyzers:**
- Improve FastAPI route detection (handle more decorator patterns)
- Improve Django URL pattern parsing (nested routers, include())
- Better detection of SQLAlchemy model relationships
- Detect more file roles (middleware, dependency, schema, celery task)
- Handle edge cases: syntax errors in user files, empty files, binary files

**MCP Tools:**
- Improve tool docstrings (Claude reads these to understand what to call)
- Add filtering params to existing tools (e.g. filter call graph by file role)
- Improve error messages returned to the AI client
- Add a new MCP tool if a clear gap exists

**Tracer:**
- Improve the dynamic tracer output format
- Handle more entry point types (class methods, async generators)
- Better error reporting when the traced function fails

**Storage:**
- Add a missing query function to database.py
- Improve UPSERT logic for a table
- Add an index for a frequently queried column

**API:**
- Improve a REST endpoint response format
- Add a missing endpoint the frontend needs
- Better error handling and status codes

---

## Rules

- Only edit files inside `mcp_server/` and `web_ui/api/`
- Do not touch frontend files
- All new functions must have type hints
- Do not break existing MCP tool signatures — the AI client depends on them
- Handle exceptions gracefully — return `{"ok": false, "error": "..."}` not stack traces
- Keep Python version compatible with 3.11+

---
